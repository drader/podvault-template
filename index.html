Problem 01 — Insights evaporate

An idea heard once
and never written down is lost.

The situation

Podcast listening is passive by design. You're driving, running, cooking. Even when you mean to note something, the moment passes. Even when you do note it, the note sits in a list with no context, no links, no way to find it six months later.

The fix

Drop a transcript — from your podcast app, from Whisper, from anywhere — into wiki/raw/transcripts/. The researcher agent extracts key insights, notable quotes, and resources mentioned, and writes a structured summary to wiki/episodes/. Searchable via MCP hybrid search from any Claude session.

Problem 02 + 03 — Guest context and topic silos

Guests and topics
get a page of their own.

The situation

A researcher appears on three shows with different hosts over two years. Their thinking evolves. Episode 1 contradicts episode 3. You have no way to see the arc — each episode is a closed file. Topics discussed across shows have no cumulative record.

The fix

Every ingest creates or updates wiki/guests/ and wiki/topics/ — each guest page linking to every episode they appear in, each topic page aggregating insights across shows. Guest profiles, host profiles, show profiles — all cross-linked bidirectionally.

Problem 04 — Patterns stay invisible

The Monday orchestrator
surfaces what you missed.

The situation

Four guests recommend the same book in four different episodes across 18 months. Seven episodes mention the same framework from different angles. These patterns only become visible to someone who has read everything — and nobody re-reads episode notes.

The fix

Every Monday the orchestrator scans agent memory for topics appearing across 2+ episodes, promotes them to wiki/syntheses/, and writes a "topics worth exploring" note to the journal. Single-mention topics that appear repeatedly get flagged for promotion.
